An approximation algorithm for the TSP problem, using a DFS algorithm. There is a high likelihood of false negatives, especially in graphs with very few edges.
Complexity: O(V + E)) => O(V^2)

Parameters
gThe graph on which to perform the algorithm
startVertexThe vertex on which the path starts
resultLengthThe length of the tour found. This result will not yield conclusive information if the return value is false.
Returns
True if a tour containing all vertices found, false otherwise

An approximation algorithm for the TSP problem, using Dijkstra's algorithm. There is a high likelihood of false negatives. It doesn't work well when the starting vertex has a direct edge to a lot of other vertices.
Complexity: O(V ^ 3)

Parameters
gThe graph on which to perform the algorithm
startVertexThe vertex on which the path starts
resultLengthThe length of the tour found. If no path is found, this yields the length of a tour containing nonexistent edges.
Returns
True if a tour containing all vertices found, false otherwise
